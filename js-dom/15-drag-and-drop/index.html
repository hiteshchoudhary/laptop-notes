Example 1 — HTML Drag and Drop API Overview

Native browser feature. Works through events on both dragged element (source) and drop zone (target).

Source Events

1. dragstart — setup data transfer
2. drag — fires repeatedly while moving
3. dragend — after drop (always fires)
→

Drop Zone Events

1. dragenter — element enters zone
2. dragover — hovering over zone
3. dragleave — leaves zone
4. drop — user releases

KEY TAKEAWAYS

        
    
          
  1. Drag events: SOURCE (dragstart, drag, dragend) and DROP ZONE (dragenter, dragover, dragleave, drop).
    2. 
          
  2. CRITICAL: preventDefault() in dragover is required for drop to work.
    3. 
          
  3. dataTransfer.setData() in dragstart, getData() in drop only. Data is unreadable in dragover (security).
    4. 
          
  4. effectAllowed (dragstart) controls available operations. dropEffect (dragover) controls cursor.
    5. 
          
  5. Sortable lists: getBoundingClientRect() to find insertion point by comparing mouse Y to card centers.
    6. 
          
  6. dragleave fires on child elements — check relatedTarget or contains().
    7. 
          
  7. HTML DnD API does NOT work on touch. Use pointer events as fallback.
    8. 
          
  8. Accessibility: keyboard alt (Space=pick/drop, Arrows=move, Escape=cancel) with ARIA live regions. REQUIRED by WCAG 2.1.
    9. 
          
  9. dataTransfer.files provides desktop file drag support. Always validate type and size.
    10. 
          
  10. For production, consider SortableJS or dnd-kit. But understanding native API is essential.
