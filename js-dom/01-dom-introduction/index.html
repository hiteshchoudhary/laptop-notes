Example 1 — The Flipkart Blueprint: What is the DOM?

The DOM is a tree-structured representation of an HTML document. Every tag becomes an Element node, every piece of text becomes a Text node, and even comments become Comment nodes.

Example 2 — window vs document vs globalThis

Think of a web app as an office building. window is the entire building. document is the main hall with HTML content. globalThis is a universal key for any JS environment.

Example 6 — DOM vs HTML Source

      
The DOM is a LIVE object. The HTML source is STATIC text. The browser auto-corrects, JavaScript modifies, and user input changes the live DOM.

Example 7 — console.dir() vs console.log()

      
console.log(element) shows HTML markup. console.dir(element) shows the JavaScript object with all properties.

Example 11 — Common Gotchas

      
The top 5 DOM mistakes new developers make.

      

        

          1
          Script in <head> without defer → document.body is null
        

        

          2
          childNodes includes whitespace text nodes; use children instead
        

        

          3
          getElementsBy* is LIVE; querySelectorAll is STATIC
        

        

          4
          innerHTML += destroys event listeners on existing children
        

        

          5
          Reading offsetHeight in a DOM-writing loop = layout thrashing
        

      

      
      

    


    
    

      
Example 12 — DOMContentLoaded vs window.onload

      
DOMContentLoaded fires when HTML is parsed. window.onload fires after ALL resources load. Most scripts use DOMContentLoaded.

      

    


    
    

      
Key Takeaways

      
    
        
  1. The DOM is a LIVE tree of objects, not the same as HTML source.
    2. 
        
  2. window = global browser object; document = HTML document; globalThis works in all JS environments.
    3. 
        
  3. Every node has nodeType (1=Element, 3=Text, 8=Comment), nodeName, and nodeValue.
    4. 
        
  4. document.documentElement, .head, .body are instant shortcuts.
    5. 
        
  5. console.dir(element) shows the JS object; console.log() shows HTML.
    6. 
        
  6. Rendering pipeline: HTML → DOM → CSSOM → Render Tree → Layout → Paint → Compositing.
    7. 
        
  7. DOM manipulation is expensive: batch changes, use DocumentFragment, prefer CSS transforms for animations.
    8. 
        
  8. Use .children over .childNodes unless you need text/comment nodes.
    9. 
        
  9. Use DOMContentLoaded or <script defer> to ensure DOM is ready before running your code.
